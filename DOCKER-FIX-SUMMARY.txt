ROOT CAUSE:
===========
- Docker node:18-alpine doesn't support import.meta.dirname in tsx
- TWO files used import.meta.dirname: vite.config.ts AND server/vite.ts
- vite.config.ts caused container restart loop
- server/vite.ts caused Web UI TypeError on page load
- Extracted tar.gz files don't preserve execute permissions in Docker

SOLUTION:
=========
- Added chmod +x to set execute permissions on startup script
- Startup script automatically replaces import.meta.dirname with process.cwd()
- Patches BOTH vite.config.ts AND server/vite.ts for complete compatibility
- Provides Docker-compatible path resolution in all files
- No manual file editing required
